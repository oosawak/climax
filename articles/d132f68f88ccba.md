Climax Chronicle の主目的は **セッション管理** と **ログ管理**です。

この記事は「いま動いている構成」に合わせて、**ログを貯める → ブラウザで見る**までの最小手順をまとめます。

## 全体像

```text
tmux（cmd/log/codex セッション）
  ↓（コマンド実行単位ログ）
clients/ctmcmd.py
  ↓  POST /api/log/append
Azure Functions (Chronicle API)
  ↓（永続化）
Cosmos DB
  ↑  GET /api/sessions, /api/logs, /api/logs/recent など
GitHub Pages（管理画面: docs/admin）
```

ポイント:

- ログは “ターミナル全文” ではなく、まずは **コマンド実行単位**で十分（topicで分類）
- 管理画面は **GitHub Pages** で公開（静的ホスト）
- API は **Azure Functions**（Function Key で保護）

## 0) 前提（用意する値）

- `CLIMAX_FUNCTIONS_URL`
  - 例: `https://func-api-eedplxgcbbmra.azurewebsites.net`
- `CLIMAX_FUNCTIONS_CODE`
  - Azure Functions の **default function key**

## 1) CLI 側：ログを貯める

### 1-1. 初回セットアップ

```bash
./clients/setup_clients_env.sh
```

生成される `clients/.env` に、少なくとも以下を設定します（例）:

```bash
CLIMAX_FUNCTIONS_URL="https://func-api-eedplxgcbbmra.azurewebsites.net"
CLIMAX_FUNCTIONS_CODE="<配布された function key>"
```

### 1-2. tmux セッション管理（覚えない）

```bash
./clients/ctm
```

`ctm` はメニューから `start` / `cmd` / `log` を選ぶ想定です。

### 1-3. コマンド実行ログを送る

cmd セッション（作業セッション）で、コマンドを `ctmcmd.py` 経由で実行します。

```bash
./clients/ctmcmd.py -- echo hello
```

topic を固定したい場合は `clients/.env` に:

```bash
CLIMAX_TOPIC="build"
```

## 2) Web 側：管理画面（GitHub Pages）で見る

### 2-1. 公開するファイル

このリポジトリでは GitHub Pages 用の管理画面を `docs/` に置きます。

- `docs/index.html`（`docs/admin/` へ誘導）
- `docs/admin/index.html`（UI）
- `docs/admin/main.js`（API呼び出し）
- `docs/admin/style.css`

### 2-2. GitHub Pages の設定

GitHub のリポジトリ設定で Pages を有効にします。

1. Repo Settings → Pages
2. Source: “Deploy from a branch”
3. Branch: `main`
4. Folder: `/docs`

### 2-3. 管理画面の入力

管理画面（`/admin/`）を開いたら、Connection セクションを埋めます。

- `API base`
  - `https://<functionapp>.azurewebsites.net`
  - 末尾の `/api` は省略OK（画面側で補完）
- `Function code`
  - `CLIMAX_FUNCTIONS_CODE`（`?code=` の中身）
  - `?code=...` の形で貼ってもOK
- `server_id`
  - 例: `i9`

その後:

- `Recent logs`（候補が出る）
- `Load logs`（ログ一覧）
- `Load sessions`（セッション一覧）

## 3) CORS 設定（GitHub Pages → Functions を許可）

GitHub Pages から Azure Functions を呼ぶには、Functions 側に CORS 設定が必要です。

このリポジトリには設定用スクリプトがあります。

```bash
cd ~/Workspace/climax
AZ_RESOURCE_GROUP=rg-climax \
AZ_FUNCTION_APP=func-api-eedplxgcbbmra \
bash scripts/az_set_functions_cors.sh
```

環境に合わせて `AZ_RESOURCE_GROUP` / `AZ_FUNCTION_APP` を変更してください。
